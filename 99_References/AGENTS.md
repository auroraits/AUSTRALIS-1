# AGENTS.md — 99_References (Fuentes y referencias)

> **Hereda:** todas las reglas del `AGENTS.md` raíz (`/AGENTS.md`). Este archivo solo agrega reglas locales del subsistema.
>
> **Rol:** soporte informativo, no normativo. Las referencias aquí **no** toman precedencia sobre ADRs ni `00_MVP/MVP v2.2.md`.

## Propósito
Repositorio de referencias:
- proveedores, standards, papers, links, herramientas, datasets.

No es “decisión”: es soporte.

## Cómo debe trabajar un agente aquí
- Mantener entradas con:
  - título, link, fecha consultada,
  - breve nota “por qué importa”,
  - a qué subsistema aplica.
- No duplicar referencias: preferir consolidar y linkear desde docs.
- Si una referencia respalda una decisión, el ADR debe linkearla.

## Estructura recomendada
- Suppliers.md (por región / disponibilidad)
- Standards.md
- Research_Links.md
